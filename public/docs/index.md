# Documentation Index

Use this map to find the repository guide that matches the task. Keep policy
terms aligned with `public/docs/policy-vocabulary.md`.

## Start Here

- `README.md`: reviewer quick start, environment setup, and top-level project
  layout.
- `AGENTS.md`: repository rules for coding agents and scoped handoffs.
- `CONTRIBUTING.md`: short contributor checklist for small patches and pull
  requests.

## Build And Validation

- `public/docs/report-builds.md`: report build wrapper, validation-only builds,
  artifact-refresh builds, consumed-input tracking, and failure summaries.
- `public/docs/ops-tooling.md`: packaged Python `ops-*` commands for
  experiments, audits, renderers, orchestration, and evidence summaries.
- `public/docs/julia-cli-workflows.md`: Julia command families exposed through
  `packages/julia/bin/stenosis-hemodynamics`.

## Artifacts And Data

- `public/docs/artifact-policy.md`: artifact classes, non-negotiable cleanup
  rules, and baseline validation gates.
- `public/docs/report-assets-and-provenance.md`: report asset directories,
  owning commands, TeX consumers, and refresh validation.
- `public/docs/resolved3d-workflows.md`: optional resolved-3D inputs, skip
  behavior, comparison workflows, and publication boundaries.
- `public/docs/benchmark-pipeline.md`: package benchmark profiles, outputs, and
  report-asset publication rules.

## Handoffs And Release

- `public/docs/agent-workflows.md`: `ops-orchestrate` modes, profiles, packet
  checks, and validation defaults.
- `public/docs/publication-readiness.md`: public export rules and release
  validation.
- `public/docs/policy-vocabulary.md`: modal verbs and shared artifact/build
  vocabulary.
- `public/docs/executive-assessment.md`: dated manuscript assessment; do not
  treat it as current workflow policy.
