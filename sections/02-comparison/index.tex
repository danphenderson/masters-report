\section{Preliminary Resolved-Velocity Diagnostic at \texorpdfstring{$T=1$}{T=1}}
\label{sec:resolved-3d-comparison}

This section reports a numerical comparison between the stenosis-aware 1D
area--flow model and the available resolved 3D velocity data. The comparison is
an output-operator check: it is not clinical pressure evidence, nor is it an
exact cross-section quadrature of the 3D tetrahedral solution. The completed
diagnostic averages node-centered axial velocity over thin axial slabs and
compares those values with the interpolated 1D mean velocity $Q/a$, where
$a=R^2$ is the solver coordinate.
Radial profiles use the same slab nodes binned by
$\sqrt{x^2+y^2}/R_0(z)$ and compare them with the 1D closure profile.
The tables and figures in this section are baseline Newtonian diagnostics;
non-Newtonian rheology descriptors require separately generated comparison
datasets.

All numbers below come from the comparison artifact
\path{simulations/output/3d_comparison/full_t1_native_nx400/}, interpreted
under Convention~\ref{conv:computed-realization-record}. The regeneration
command, source-path convention, and file hashes are listed in
Appendix~\ref{app:code-and-ai-use}. The comparison uses baseline parameters
with final time
$T=1.0$ s and the geometry-rest initial condition, with $N=400$
finite-volume cells on $L=6$ cm
($\Delta z=0.015$ cm), the native SSPRK3 time integrator, MUSCL finite volume
with the minmod limiter, timestep cap $10^{-5}$ s, and CFL cap $0.45$.
The initial state is geometry-rest, $a=R_0^2$ and $Q=0$. The inlet boundary
uses the parabolic-profile mean velocity
$\bar u_{\mathrm{in}}=0.5(45)=22.5$ cm/s and imposes
$Q_{\mathrm{in}}=R_0(0)^2\bar u_{\mathrm{in}}$; the outlet fixes
$a_{\mathrm{out}}=R_0(L)^2$ and computes the ghost flow from the outgoing
Riemann invariant. The two 3D source identifiers are the default upstream XDMF
paths \path{simulations/data/3d/canic_case3/77/velocity.xdmf} and
\path{simulations/data/3d/canic_case3/60/velocity.xdmf}, with severities
23\% and 40\%, target time $1.0$ s, and tolerance $10^{-3}$ s. The matched
XDMF time for both included cases is $0.9994999999999453$ s. Section means
use 200 equally spaced axial targets and slab half-width
$\max(\Delta z_{\mathrm{sec}}/2,0.015)=0.0150753769$ cm; radial profiles use
the declared slices $z=1.951,2.451,2.951$ cm and 20 radial bins in
$\sqrt{x^2+y^2}/R_0(z)$. The raw XDMF/HDF5 inputs are not archived in the
report package; the archived evidence available to the PDF consists of the
copied static plot data in \path{figures/static/static/data/stenosis-comparison/}
and the provenance hashes in Appendix~\ref{app:code-and-ai-use}.

In the radius-squared solver convention, the stored variable \texttt{A} is the
coordinate $a=R^2$, not the physical area $A_{\mathrm{phys}}=\pi a$. The flow
variable is therefore the corresponding $\pi$-scaled flow:
$Q_{\mathrm{phys}}=\pi Q$. All comparison velocities in this section use the
field $\texttt{velocity(result)}=Q/a$, which equals
$Q_{\mathrm{phys}}/A_{\mathrm{phys}}$ under this scaling. The plots and
tables should not be read as using the stored $Q$ divided by
$A_{\mathrm{phys}}$.

The Section~\ref{sec:resolved-3d-comparison} diagnostic uses the Canic extended
radius-squared finite-volume realization. The corresponding forward-model
descriptor is \path{canic-extended-1d}. It uses
$\mathcal W_{\mathrm{elastic1D}}$, the elastic pressure--area wall law in
Definition~\ref{def:1d-elastic-wall-law}, \texttt{newtonian} rheology
($\nu=0.04$ cm$^2$/s), and the parabolic velocity profile
$\mathcal P=\texttt{parabolic}$, so $\alpha_{\mathcal P}=4/3$ and
$g_{\mathcal P}=4$. The Canic variable-radius corrections are included: the
flux uses $\alpha_{\mathcal P}+\alpha_c(R_0')$, and the source
includes the $p_2$, $R_0'$, $R_0''$, and $\alpha_c'$ terms described in
Appendix~\ref{app:numerical-methods-details}.

The two included upstream states are case 77, treated as a 23\% stenosis, and
case 60, treated as a 40\% stenosis. Both XDMF files report
$t=0.9994999999999453$, within $5.0\times 10^{-4}$ of $T=1$. Upstream
case 50 was not used because its XDMF time is
$t=1.4994999999998904$, outside the $10^{-3}$ tolerance for the $T=1$
comparison.

Figure~\ref{fig:t1-section-mean-comparison} shows the section-mean comparison.
The 1D model captures the smooth upstream and downstream trend but overpredicts
the highest-velocity region near the stenosis, especially for the 40\% case.
The largest section errors occur for case 60 around $z=2.4$--$2.8$ cm,
where the 1D mean velocity is between 58 and 67 cm/s while the node-slab 3D
mean is between 19 and 27 cm/s.
Because the largest relative errors exceed unity near the stenosis, this comparison
does not support an accuracy claim for the 1D model in the throat region. It is
still useful as a velocity-localization diagnostic: the mismatch concentrates
where the reduced velocity closure, idealized 1D boundary state, and node-slab
output operator are least matched to the resolved 3D field.

The radial-profile diagnostics in
Figure~\ref{fig:t1-radial-profile-comparison} show the same pattern at the
profile level. For both cases, the throat slice $z=2.451$ cm has fewer
populated radial bins because fewer 3D nodes lie in the narrowest cross-section
slab. The largest radial-profile discrepancy is downstream of the throat at
$z=2.951$ cm, where case 60 reaches a maximum per-bin absolute error of
41.72 cm/s.
The dashed 1D curves use the declared velocity-profile closure evaluated from
the section mean $\bar u=Q/a$. For the power-family diagnostic,
\[
    u_{\mathcal P}(r)
    =
    \frac{\gamma+2}{\gamma}\,\bar u
    \left(1-\left(\frac{r}{R}\right)^\gamma\right),
    \qquad
    \alpha_{\mathcal P}=\frac{\gamma+2}{\gamma+1}.
\]
An $\alpha=1.1$ power-profile diagnostic corresponds to the $\gamma=9$ member of
this family; the parabolic profile is $\gamma=2$ and
$\alpha_{\mathcal P}=4/3$. Flat-profile diagnostics use
$u_{\mathcal P}(r)=\bar u$ with $\alpha_{\mathcal P}=1$.

These discrepancies should be read in light of the comparison boundary in
Section~\ref{subsec:model-hierarchy-comparison-boundary}. The 1D solver is
compared to a resolved 3D velocity field through node-slab output operators.
The result locates where the reduced velocity closure departs from the resolved
state, but it does not attribute that departure uniquely to the 1D closure,
boundary data, 3D post-processing operator, or formulation differences. In
summary, the $T=1$ comparison is a declared velocity diagnostic whose largest
deviations are localized near and downstream of the stenosis throat, especially
for the 40\% case. It should not be converted into a pressure, clinical, or
resolved-3D validation claim. The next defensible study step is therefore a
matched comparison dataset that restores the raw 3D inputs, archives the source
hashes or upstream release identifiers, and reruns the same output operators
while varying only one declared closure or boundary choice at a time.

\subsection{Package Benchmark Suite}
\label{subsec:package-benchmark-suite}

The report package now includes a reproducible package-benchmark pipeline for
the Julia solver package and Python CLI. The suite is an internal evidence
record for self-convergence, backend agreement, descriptor smoke/health,
runtime comparison, and resolved-velocity output-operator diagnostics within
the emitted CSVs. It does not upgrade those checks into clinical validation,
pressure or FFR accuracy, full transient FEM validation, or Julia--Python
scientific equivalence beyond the metrics actually compared.

\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
./scripts/julia-release simulations/run_package_benchmark.jl \
  --profile smoke \
  --output-dir simulations/output/package_benchmark/<run_id> \
  --overwrite \
  --include-python \
  --publish-report-assets

pipenv run python scripts/render_package_benchmark_figures.py \
  --benchmark-dir simulations/output/package_benchmark/<run_id>
\end{lstlisting}

The smoke profile is a deterministic wiring check. The overnight profile expands
the same schemas to the descriptor-health, refinement, native/SciML parity,
stationary-Stokes projection, rheology/profile, OpenBF-style boundary,
resolved-velocity, and Python CPU/Torch-MPS comparison stages. Optional SciPy
and resolved-3D inputs are treated as availability boundaries: absent inputs
produce structured skipped rows rather than manuscript claims.
