\section{Implementation Checks and Targeted Numerical Verification}
\label{sec:numerical-verification}

This chapter reports the implementation checks and targeted numerical
verification now available for the selected solver. The title is intentionally
narrower than ``full numerical verification.'' The evidence below supports the
finite-volume implementation, diagnostics, data contracts, and selected
verification problems used in this report. It does not establish clinical
validation, pressure or FFR accuracy, or exhaustive verification of every
possible model variant.

\subsection{Software and Descriptor Checks}
\label{subsec:verification-reproducibility}
\label{subsec:package-benchmark-suite}

The reproducible benchmark pipeline records implementation-check stages for the Julia
solver: descriptor health, sampled self-convergence, native/SciML
cross-integrator discrepancy, stationary-Stokes projection rows,
rheology/profile sensitivity, OpenBF-style boundary handling, and
resolved-velocity output diagnostics. These stages check that the selected
implementation surface remains runnable and schema-stable. Descriptor-health
rows exercise the report-now spatial surface from
Table~\ref{tab:implemented-discretization-surface}: first-order finite volume,
MUSCL finite volume, Richtmyer/Lax--Wendroff finite volume, and modal DG
degrees $p=0,1,2$, paired with the native Euler, SSPRK2, and SSPRK3 steppers.
Refinement rows provide FV/DG spatial context, while backend-parity rows compare
native SSPRK3 against the supported SciML/OrdinaryDiffEq policies for
semi-discrete operators admitted by that backend.

\subsection{Manufactured-Solution Verification}
\label{subsec:mms-verification}

The implementation now exposes an opt-in manufactured forcing term. The
production equations use \texttt{NoForcing()} by default. For verification
only, \texttt{ManufacturedForcing()} supplies a smooth exact state
$(a^\star(z,t),q^\star(z,t))$, exact initial data, exact boundary states, and
source terms inserted into the existing finite-volume right-hand side. The
runner
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
./scripts/stenosis-hemodynamics verify mms \
  --nxs 50,100,200,400 \
  --dt-values 2e-5,1e-5,5e-6,2.5e-6 \
  --output-dir figures/static/static/tables/verification \
  --overwrite
\end{lstlisting}
emits raw discrete $L^1$, $L^2$, and $L^\infty$ errors, with observed orders
computed from the $L^2$ rows, for spatial refinement and requested-timestep
sensitivity. The native backend also records realized CFL diagnostics because
the accepted timestep is
$\min(\Delta t_{\mathrm{request}},\Delta t_{\mathrm{CFL}})$. These rows are
method-of-manufactured-solutions evidence for the implemented operator; the
temporal rows should not be read as a clean temporal-order estimate when the
spatial error floor or CFL cap dominates. They are not a physical validation
case.

\subsection{Zero-Forcing Geometry-Rest Drift}
\label{subsec:rest-state-drift}

The continuous model has a geometry-rest equilibrium at
$a_i=R_0(z_i)^2$, $q_i=0$ under zero forcing. The discrete Rusanov/MUSCL
operator is not exactly well balanced for spatially varying $R_0$ because the
mass flux includes numerical viscosity. The verification runner therefore
starts the selected C23 and C40 geometries from the rest state and reports
\[
    \max_i |q_i|,\qquad
    \max_i |a_i-R_0(z_i)^2|,\qquad
    \left|\sum_i a_i\Delta z-\sum_i a_i(0)\Delta z\right|
\]
as functions of grid size and elapsed time. This test is required context for
any comparison run initialized from the same geometry-rest state.

\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
./scripts/stenosis-hemodynamics verify rest \
  --severities 23,40 \
  --nxs 100,200,400,800 \
  --elapsed-times 0,0.001,0.01,0.1 \
  --output-dir figures/static/static/tables/verification \
  --overwrite
\end{lstlisting}

\subsection{Boundary and Conservation Diagnostics}
\label{subsec:boundary-diagnostics}

The native backend now persists per-run diagnostics in
\texttt{SimulationDiagnostics}: accepted timestep extrema, realized CFL extrema,
$\lambda_-$ and $\lambda_+$ extrema, the minimum subcriticality margin
$\min(-\lambda_-,\lambda_+)$, mass extrema and mass defect, positivity
projection count, and total area correction from the positivity floor. These
diagnostics are emitted by direct simulation summaries, benchmark case rows,
rest-state drift rows, and resolved-velocity comparison summaries.

The fixed-area characteristic boundary rule remains a method applied under the
subcritical sign condition $\lambda_-<0<\lambda_+$. A positive characteristic
radicand is not enough. The persisted sign extrema and subcriticality margin
are therefore the relevant evidence for the boundary regime in each accepted
native run.

\subsection{Stationary-Stokes Refinement}
\label{subsec:stationary-stokes-projection-checks}

The stationary-Stokes module now emits a refinement summary rather than only a
projection-existence check. It compares projected one-dimensional section
averages with finite-element section samples across generated stationary
Stokes meshes and reports finest-mesh relative discrepancies. This remains an
initializer/projection diagnostic; it is not transient 3D validation of the 1D
solver.

\subsection{Verification Summary}
\label{subsec:verification-summary}

The strengthened record supports a targeted numerical-verification claim. It
includes exact-solution MMS evidence, zero-forcing rest-state drift diagnostics,
persisted CFL and characteristic-speed metadata, mass-balance and positivity
metadata, sampled self-convergence, cross-integrator checks, and
stationary-Stokes projection refinement. The claim remains bounded: runtime,
platform, broad rheology/profile sensitivity, Lax--Wendroff rows, and
nonselected DG rows are implementation-check or appendix-level context unless
tied to a principal research question.
