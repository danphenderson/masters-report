\section{Code Availability and Build Statement}
\label{app:code-and-ai-use}

This report is built from the \LaTeX{} sources rooted at
\texttt{final-report.tex}. The compiled PDF should be treated as the rendered
form of those sources. The report package used for this build is the repository
root containing \path{final-report.tex}. Because no public remote or archival
release is declared here, this appendix does not claim a public repository or
archival URL.

Julia project. The tracked source inventory is:

The standard package import surface used by report regeneration commands is:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true,language=julia]
using CanicExtended1D

run_available_resolved3d_comparison(
    output_dir="simulations/output/3d_comparison/full_t1_native_nx400",
    overwrite=true,
)
\end{lstlisting}

The package-benchmark suite is regenerated separately from the tracked PDF. A
smoke run writes deterministic schema checks; an overnight run expands the same
schemas to the full benchmark matrix:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
./scripts/julia-release simulations/run_package_benchmark.jl \
  --profile smoke \
  --output-dir simulations/output/package_benchmark/<run_id> \
  --overwrite \
  --include-python \
  --publish-report-assets

pipenv run python scripts/render_package_benchmark_figures.py \
  --benchmark-dir simulations/output/package_benchmark/<run_id>
\end{lstlisting}

The Section~\ref{sec:resolved-3d-comparison} comparison artifact is the
non-archived directory
\path{simulations/output/3d_comparison/full_t1_native_nx400/}. The command
shape for regenerating the comparison, when the raw 3D inputs are restored, is:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
./scripts/julia-release -e 'using CanicExtended1D; run_available_resolved3d_comparison(output_dir="simulations/output/3d_comparison/full_t1_native_nx400", overwrite=true)'
\end{lstlisting}
