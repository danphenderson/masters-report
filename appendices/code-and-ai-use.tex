\section{Code Availability and Release Record}
\label{app:code-and-ai-use}

This report is built from the \LaTeX{} sources rooted at
\path{final-report.tex}. The solver package is the root Julia package
\texttt{StenosisHemodynamics}, with package identity in \path{Project.toml},
resolved Julia dependencies in \path{Manifest.toml}, and the package entry point
at \path{src/StenosisHemodynamics.jl}. The command-line surface used by the
report is \path{scripts/stenosis-hemodynamics}, which dispatches through
\path{scripts/stenosis-hemodynamics.jl} and the root Julia project.

The D16 proposal base commit is
\texttt{beb774d8d32addf4fd64fae722b51ce106b9671f}. A final public release commit
or tag is not assigned before this proposal is integrated. The release gate must
replace the placeholder \texttt{D16-INTEGRATION-RELEASE-ID} in this appendix and
in \path{editorial/release/D16-release-manifest.json} with the final integrated
commit or tag before publishing a final PDF.

\paragraph*{Build command.}
The report build command is:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
latexmk -pdf -interaction=nonstopmode -halt-on-error -outdir=/tmp/masters-report-build final-report.tex
\end{lstlisting}
Julia package tests are run through the repository launcher:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
./scripts/julia-release test/runtests.jl
\end{lstlisting}

\paragraph*{Comparison-asset regeneration command.}
When the optional raw 3D inputs are restored locally, the tracked comparison
assets are regenerated with:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
./scripts/stenosis-hemodynamics compare-3d \
  --data-root simulations/data/3d/canic_case3 \
  --output-dir simulations/output/3d_comparison/full_t1_native_nx400 \
  --nx 400 \
  --target-time 1.0 \
  --time-atol 1e-3 \
  --overwrite \
  --publish-report-assets \
  --report-assets-dir figures/static/static/data/stenosis-comparison \
  --radial-bin-counts 10,20,40 \
  --radial-radius-modes current,reference
\end{lstlisting}
Package-benchmark assets are regenerated separately from the PDF. Use the
\texttt{benchmark} subcommand of \path{./scripts/stenosis-hemodynamics}; render
those assets with \path{scripts/render_package_benchmark_figures.py}. Those rows
are secondary implementation-health records, not the principal comparison
method.

\paragraph*{Raw 3D inputs and manifest.}
The raw upstream XDMF/HDF5 velocity inputs used for
Figure~\ref{fig:stenosis-3d-reference-geometry} and
Section~\ref{sec:resolved-3d-comparison} are not archived in this report
package. The expected local input root is:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
simulations/data/3d/canic_case3/
\end{lstlisting}
When available, that directory is restored from this upstream repository,
commit, and subtree:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
https://github.com/qcutexu/Extended-1D-AQ-system
056a9da2b36b480691f18025d242d2c00f6e7180
case3_all_3d_results/
\end{lstlisting}
The report archives generated CSV and PGFPlots-ready static assets, not the raw
fields. Current/deformed 3D geometry, displacement application before plane
cuts, wall/boundary/material histories, and exact-time 0.9995 s resampling
remain unresolved unless a later release record adds those data.

The file-hash record is stored outside the thesis PDF. Its manifest path is:
\begin{lstlisting}[basicstyle=\ttfamily\footnotesize,breaklines=true]
editorial/release/D16-release-manifest.json
\end{lstlisting}
Detailed SHA-256 values live there or in another tracked manifest, not as large
tables inside this appendix.
