\medskip
\noindent\textbf{Abstract.}\par\smallskip
Hemodynamic results are difficult to compare across resolved, reduced, lumped,
multiscale, and learned model classes because each class changes the state
variables, closure assumptions, boundary data, observables, and evidence needed
to support a claim. This report develops a mathematical reading framework for
blood-flow simulation, using stenotic flow as the organizing example. The
framework separates model tier from evidentiary strength: a pressure, flow,
velocity, WSS, or FFR-style quantity becomes interpretable only after the
governing state, closure assumptions, boundary approximation, discretization,
observation map, and metric have been specified.

The review compares resolved 3D CFD/FSI descriptions, axisymmetric and
two-dimensional reductions, distributed one-dimensional area--flow systems,
lumped networks, geometrical multiscale couplings, and learned surrogate or
operator maps. It then applies the framework to an
idealized stenosis case study. The case study specifies a reduced
one-dimensional model, finite-volume realization, manufactured-solution
verification check, geometry-rest preservation test, and plane--tetrahedron
observation operator for comparison with available resolved velocity data.

The case study is methodological. It provides implementation-verification
evidence for the forced operator, identifies a non-well-balanced rest-state
limitation of the reported discretization, and quantifies final-time
section-mean velocity discrepancies under a specified 1D--3D observation map.
It does not establish external clinical validation, pressure-ratio validation,
or equivalence to a fully matched transient FSI computation. The central
conclusion is that model dimension is not an accuracy ladder: each model tier
changes the state space, closure burden, native observables, and evidence
required to support a hemodynamic claim.
